# 日常使用

开发新功能时先创建一个新的分支，一般都是开发一个新功能，新建一个新分支

  1. 用来创建并跳转到新分支
    git checkout -b BranchName

  2. 查看当前现有的分支
    git branch

  3. 跳转分支
    git checkout BranchName

  4. 开发完后提交分支
    git push -u origin BranchName

（加了参数 - u 后，以后即可直接用 git push 代替 git push origin master
意思就是我们不用在后面加 origin + 要提交的分支名 ，直接 git push 即可
但是前提是，第一次提交需要加 -u 参数后，后面的提交就直接可以 git push）

  1. 合并分支
    先要跳转到 master 分支上
    git checkout master
    然后
    git merge BranchName

  2. 提交主分支
    git push

  3. 在这些操作之前要
    git add .
    git commit -m"备注"

  4. 首次拉取代码
    git clone 远程仓库地址

  5. 每次开发之前要拉取远程仓库最新的代码到本地
    git pull

  6. git add .
    git commit -m"备注"
    之类的指令是对本地仓库操作，拉取、提交时是本地和远程仓库之间的联系
